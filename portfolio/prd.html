PRD 01

产品总 PRD：学术论文润色工具

这部分对应原“产品需求文档”，用于说明产品定位、用户场景、竞品分析、功能范围和核心指标。

版本
V1.0
对象
学生 / 科研写作者
范围
论文段落润色

1. 产品背景

论文写作者经常需要把初稿中的口语化、泛化表达调整为更正式的学术语言。通用聊天工具虽然可以润色，但结果不稳定，也不一定说明改动原因。第一版产品目标是把“单次润色”做成一个清晰、可复盘的工具流程。

用户目标

快速获得更正式、清晰的论文段落表达。

产品机会

在通用聊天工具和完整写作平台之间，做一个聚焦段落润色的轻量工具。

设计原则

不新增事实、不改变结论、输出可解释、结果可追踪。

2. 用户画像与场景

用户典型场景核心痛点产品响应
本科毕业生毕业论文初稿润色表达偏口语，不知道如何改成论文语言。提供学术化润色和修改说明。
研究生小论文、投稿前语言优化担心实验结论、数值或术语被改错。强调语义保持和事实约束。
科研写作新手方法、结果、讨论段落优化不知道问题出在逻辑、表达还是信息不足。输出剩余问题和写作建议。

3. 竞品对标与启发

产品主要能力启发本项目取舍
Paperpal学术写作、语法检查、改写、投稿相关辅助。学术写作工具需要围绕论文场景做深。先聚焦中文段落润色，不扩展到投稿审稿和查重。
Grammarly语法纠错、清晰度建议、语气调整、跨平台写作辅助。写作工具需要嵌入用户真实工作流。通过历史记录和修改说明增强复盘能力。
QuillBot改写、总结、语法检查、引用工具。改写工具需要提供不同风格和强度。后续增加轻度、标准、深度三档润色强度。
秘塔写作猫中文纠错、改写、续写、翻译等综合写作能力。中文场景需要本地化表达和纠错能力。保留中文论文语境，避免做成泛写作工具。
通用对话工具自由对话、文本生成、解释和改写。能力强，但输出格式和结果稳定性依赖用户提示。用固定流程、结构化结果和评分规则降低使用门槛。

4. 版本规划

V1.0 Demo 验证

论文段落润色、结构化结果、匿名历史记录、基础评估报告。

V1.1 体验优化

结果对比、历史详情、复制、重试、输入长度提示、Markdown / Word 导出。

V1.2 产品化

Prompt 版本管理、多模型对比、用户登录、免费额度、反馈入口。

5. 核心功能需求

模块需求描述规则优先级
论文润色用户输入论文段落后生成润色结果。保留原意，不新增事实，返回固定字段。P0
润色参数支持目标、强度、风格选择。先提供有限选项，避免用户配置过重。P1
结果展示展示润色文本、修改说明、剩余问题、知识点。支持复制和重新生成。P0
历史记录保存每次任务，支持查看和恢复。记录状态、时间、模型和结果。P0
效果评估基于测试集展示评分和问题备注。支持按场景筛选。P1
PRD 02

模型效果评估 PRD

这部分对应原“模型评测框架设计 PRD”，用于说明怎样判断润色效果是否可靠，而不是只看语言是否流畅。

测试集
15 条
评分
5 维度
用途
发现问题

1. 任务和能力标签

学术化表达 语义保持 事实与数值保持 术语保持 结构化输出 异常兜底 Prompt 注入防护 可解释性

2. 整体评测架构

  1. 维护标准测试集，给每条用例标注能力标签和观察点。
  2. 使用同一套提示词和模型配置运行测试用例。
  3. 按语义保持、学术表达、可读性、结构完整和风险控制评分。
  4. 将低分用例归入 Bad Case，判断问题来自输入、提示词、模型或产品交互。
  5. 根据归因结果调整提示词、输入校验、页面提示和模型选型。

3. 过程动作得分点设计

评分项满分得分规则扣分示例
输入理解10识别主题、方法、结论、指标和术语。把训练时间增加误判为效率提升。
任务遵循10只执行润色任务，不执行无关指令。用户输入注入指令后按注入内容输出。
改写动作10完成学术化、凝练化和逻辑优化。短文本被扩写成完整研究结论。
结构化返回10字段完整，前端可稳定展示。缺少修改说明或输出纯文本。

4. 完成质量分设计

30%语义保持：不改变原文含义、结论和限定条件。
30%学术表达：表达正式、客观、凝练，符合论文语境。
20%可读性：句子更清楚，逻辑更顺。
10%结构完整：结果字段完整，可被页面展示。
10%风险控制：不新增事实，不改数值，不执行无关指令。

5. 评测任务集设计

测试类型设计目的观察指标
口语化表达验证能否改为正式论文表达。是否减少口语化、保留原意。
数值事实验证准确率、样本量、响应时间等是否保持。数值是否被修改，单位是否出错。
术语密集验证术语保持和流程理解能力。术语是否保留，技术顺序是否正确。
短文本验证信息不足时是否过度补全。是否提示补充上下文，是否新增事实。
异常输入验证非论文文本或按钮报错文本的兜底能力。是否强行学术化无关输入。
Prompt 注入验证任务边界和安全性。是否拒绝无关指令并继续按润色任务处理。

6. Bad Case 归因规则

输入问题

文本过短、上下文不足、不是论文内容。

提示词问题

过度扩写、拔高贡献、没有限制事实新增。

模型能力问题

术语理解差、结构化输出不稳定。

产品交互问题

用户不知道该输入什么，无法理解低分原因。

7. 模型对比方案

多模型对比应使用同一测试集、同一评分规则、同一输出字段。比较维度包括润色质量、中文学术表达、结构化稳定性、事实安全、响应速度和调用成本。

PRD 03

历史记录与结果复盘功能 PRD

这部分对应原“历史记录与结果复盘功能 PRD”，用于说明历史记录为什么不是附属列表，而是后续质量复盘和数据沉淀的入口。

阶段
匿名记录
状态
running / success / failed
目标
可恢复

1. 功能背景

用户润色论文时通常会多次输入、比较结果和复制片段。如果没有历史记录，刷新页面后结果会丢失。对产品侧来说，没有历史数据也无法复盘模型效果、定位低分案例或评估提示词版本。

用户价值

找回历史润色结果，降低重复操作成本。

产品价值

沉淀输入、输出、状态和错误信息，用于后续质量分析。

技术价值

建立任务状态表，为账号、额度和数据隔离打基础。

2. 页面入口与信息架构

右侧最近任务

主工作台展示最近 10 条任务，包含状态、时间和原文摘要。

任务详情恢复

点击历史记录后，恢复原文、润色结果、修改说明和剩余问题。

失败任务复盘

失败任务展示错误状态和重试入口，不覆盖用户原始输入。

3. 字段设计

字段说明用途优先级
task_id任务唯一编号。查询和问题定位。P0
input_text用户输入的原始论文段落。恢复输入和结果对比。P0
result_json润色结果、修改说明、剩余问题等结构化数据。前端展示和后续评估。P0
statusrunning、success、failed。展示任务状态和失败兜底。P0
model_name实际调用的模型。效果复盘和成本分析。P0
prompt_version提示词版本。A/B 测试和回归评测。P1
score人工或自动评估分数。低分案例分析。P1
created_at任务创建时间。历史列表排序。P0

4. 核心交互流程

新建润色任务

提交原文后创建 running 任务，模型返回后更新为 success 或 failed。

查看历史任务

点击任务卡片，根据 task_id 获取详情并恢复结果。

复盘低分结果

为低分任务添加问题说明，将其沉淀为回归测试用例。

5. 状态流转

状态触发条件页面展示后续动作
running用户提交任务，后端已创建记录。显示生成中。等待模型返回或超时。
success模型返回并成功解析结构化结果。展示润色结果和历史卡片。允许复制、查看详情、加入评估。
failed模型失败、超时、解析失败或数据库写入失败。展示失败提示，保留原文。允许重试，记录错误类型。

6. 异常与兜底

异常场景用户提示系统处理
输入为空请先输入需要润色的论文段落。不创建任务，不调用模型。
输入过短当前文本较短，建议补充上下文。允许继续，但提示信息不足风险。
模型调用失败模型服务暂时不可用，请稍后重试。任务记为 failed，保存错误类型。
结构化解析失败结果格式异常，请重新生成。保留原始输出用于排查。
历史记录加载失败历史记录暂时无法加载，不影响继续润色。主流程可继续，记录接口错误。

7. 指标与后续迭代

历史记录点击率
历史任务恢复率
结果复制率
失败任务重试率
任务成功率
低分案例标记率
