PRODUCT REQUIREMENTS

Paper Polish Agent 产品需求文档

面向学生、研究生和科研新手的 AI 论文润色与评估工具。产品核心不是简单“帮用户改一句话”，而是提供可解释润色、历史追踪、效果评估和后续写作学习反馈。

BACKGROUND

1. 产品背景

论文写作用户常见问题包括表达口语化、逻辑不够清晰、AI 修改原因不可解释、历史版本难追踪。通用大模型虽然能润色，但输出结构不稳定，也缺少面向论文场景的质量评估闭环。

用户目标

快速把论文段落改得更学术、更清晰，同时理解修改原因。

产品机会

在通用 AI 和传统纠错工具之间，做一个聚焦中文论文段落的结构化润色工具。

设计原则

不新增事实、不改变结论、输出可解释、结果可追踪、效果可评估。

USERS

2. 用户画像与场景

用户 典型场景 核心痛点 产品机会
本科毕业生 毕业论文初稿润色 表达口语化，不知道如何改成论文语言 提供学术化润色和修改解释
研究生 小论文、投稿前语言优化 担心 AI 改变实验结论或数值事实 强调语义保持和事实约束
科研新手 实验方法、结果分析段落优化 逻辑关系不清，术语表达不规范 提供逻辑优化、术语保持和问题提示
非母语写作者 中英双语论文表达修正 翻译腔明显，学术表达不自然 后续支持双语润色和投稿表达

COMPETITIVE ANALYSIS

3. 竞品对标与启发

竞品不是为了照搬功能，而是帮助明确产品边界。Paper Polish Agent 的差异化方向是中文论文段落、可解释润色、历史追踪和质量评估。

产品 核心能力 启发 本产品取舍
Paperpal 学术写作、语法检查、改写、翻译、AI Review、查重等 学术写作工具要覆盖从初稿到投稿的流程 先聚焦中文段落润色和解释，不做完整投稿平台
Grammarly 跨平台写作助手、草稿到终稿、重写和清晰度建议 写作工具要嵌入真实工作流 后续支持 Word/Markdown 导出和历史版本
QuillBot 改写、语法、总结、引用等写作套件 改写工具需要多种风格和强度 增加轻度/中度/深度润色与风格选择
秘塔写作猫 中文纠错、改写润色、续写、翻译等 中文场景需要本土化表达和纠错能力 优先优化中文论文表达，不做泛写作工具
ChatGPT 通用对话、自由生成、灵活改写 通用模型能力强，但流程和输出不稳定 用固定结构、历史记录和评估体系形成产品差异

VERSION SCOPE

4. 版本规划

V1.0 Demo 验证

论文段落润色、润色需求选择、结构化结果、匿名历史记录、PostgreSQL 存储、评估报告页。

V1.1 体验优化

最小输入长度提示、复制按钮、历史详情对比、润色强度、风格选择、Markdown/Word 导出。

V1.2 AI 产品化

Prompt 版本管理、多模型对比、质量评分、Badcase 反馈、用户登录、免费额度。

FEATURE REQUIREMENTS

5. 核心功能需求

模块 需求描述 规则 优先级
论文润色 用户输入论文段落，选择需求后生成结构化润色结果 保留原意，不新增事实，返回固定字段 P0
润色参数 支持润色需求、润色强度、表达风格选择 强度分轻度/中度/深度；风格分学术严谨/简洁凝练/投稿表达 P1
结果展示 展示优化后文本、原文、修改明细、剩余问题、AI 知识点 支持复制、重新生成、查看解释 P0
历史记录 保存每次任务，支持查看和恢复历史结果 记录模型、Prompt 版本、状态、时间、结果 JSON P0
结果对比 对比原文和润色后文本，突出主要修改点 先做左右对比，后续再做高亮 diff P1
评估报告 基于测试集展示模型表现、低分案例和优化方向 支持按场景筛选，自动计算平均分 P1

AI REQUIREMENTS

6. AI 相关需求

Prompt 约束

保留原文核心含义，不新增事实，不改变结论，输出固定 JSON 字段，对信息不足文本给出提示。

模型评估

维护测试用例、记录 AI 输出、人工评分、查看平均分和低分 Badcase。

异常兜底

空输入不可提交，短文本提示补充上下文，模型失败记录 failed，JSON 解析失败提示重试。
