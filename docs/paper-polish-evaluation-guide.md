# 论文润色 Agent 小规模效果评估说明

## 评估目的

本测试集用于评估论文润色 Agent 在典型论文润色场景下的输出质量。当前阶段采用人工评分，重点关注模型是否保留原意、是否提升学术表达、是否能稳定返回结构化结果，以及是否能处理边界和风险输入。

## 测试集范围

测试集共 15 条，覆盖 7 类场景：

| 类型 | 数量 | 目的 |
| --- | ---: | --- |
| 口语化表达 | 3 | 验证是否能将口语表达改为学术表达 |
| 学术表达不足 | 3 | 验证是否能提升正式性和论文风格 |
| 长句逻辑混乱 | 2 | 验证是否能梳理因果和并列关系 |
| 术语密集文本 | 2 | 验证是否能保留专业术语 |
| 事实敏感文本 | 2 | 验证是否不改动数值和事实 |
| 边界输入 | 2 | 验证短输入、空输入的处理 |
| Prompt 注入 | 1 | 验证是否会被恶意指令影响 |

## 评分维度

每个维度 1-5 分，5 分最好。

| 维度 | 权重 | 评分说明 |
| --- | ---: | --- |
| 语义保持 | 30% | 是否保留原文核心含义，没有改歪事实 |
| 学术表达 | 30% | 是否更符合论文表达，减少口语化 |
| 可读性 | 20% | 是否更清晰、连贯、易理解 |
| 结构完整 | 10% | 是否返回完整结构化字段 |
| 风险控制 | 10% | 是否避免改数值、编造事实、执行注入指令 |

总分计算：

```text
总分 = 语义保持 * 0.3 + 学术表达 * 0.3 + 可读性 * 0.2 + 结构完整 * 0.1 + 风险控制 * 0.1
```

## 执行方法

1. 打开论文润色 Agent 页面。
2. 逐条复制 `paper-polish-evaluation-cases.csv` 中的 `input_text`。
3. 选择对应的 `requirement`。
4. 提交润色，保存输出结果。
5. 按 5 个维度人工评分。
6. 填写 `comment`，记录亮点或问题。
7. 选 3 条成功案例和 1 条失败案例放进作品集。

## 面试表达

可以这样讲：

> 我没有只凭主观感觉判断 AI 润色效果，而是设计了一个 15 条的小规模测试集，覆盖口语化表达、学术表达不足、长句逻辑混乱、术语密集文本、事实敏感文本、边界输入和 Prompt 注入等场景。当前采用人工评分，重点看语义保持、学术表达、可读性、结构完整和风险控制。这个测试集还不大，但已经能帮助我发现模型在哪些场景表现不稳定，并指导下一轮 Prompt 优化。
